# Contributing

This repository uses a human-reviewed, AI-assisted workflow.

## Core Expectations

- Keep changes small, focused, and reviewable.
- Prefer branch-based work with pull requests.
- Require human review before merge.
- Use AI/Codex to assist implementation, not to bypass accountability.

## Safe AI Usage

- Never include secrets, credentials, or private production data in code or prompts.
- Review AI-generated output for correctness, security, and maintainability.
- State assumptions and risks in pull requests.
- Ask for diffs before applying broad or destructive edits.

## Development Flow

1. Open or reference an issue.
2. Create a branch for one scoped objective.
3. Implement and review diffs carefully.
4. Run lint/tests locally when available.
5. Open a PR with clear summary, validation, and risk notes.

## Commit Guidance

- Use concise, imperative commit messages.
- Keep unrelated changes out of the same commit.

## Pull Request Guidance

- Explain what changed and why.
- Include validation performed.
- Highlight security/privacy impact, if any.

## Reporting Issues

- Use issue templates when available.
- Provide expected behavior, current behavior, and acceptance criteria.
